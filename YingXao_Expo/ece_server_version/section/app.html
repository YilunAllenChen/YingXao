Why Firebase?

By Yihan Jiang

Firebase Realtime Database allows us to synchronize and access a large amount of data online. In our project, we use two sub-databasesThe first one, “PlantInfo”, contains the desired moisture and sunlight requirement for a variety of plants. Our microcontroller reads these standard values from the “PlantInfo” database as a reference for controlling the sunlight shade and the water pump. The second sub-database, “Realtime-Sensor”, stores the data from real-time sensor reading. The pot-body section of the YingXao Gardener posts the real-time sensor data to the firebase, and the App section reads those sensor data as well as the desired data for the specific plant, displaying them for customers.

App Design

By Yihan Jiang

StartPage contains three buttons, which directs users to the WiFi setup page, the login page, the signup page. New users will go to the WiFi setup page first. After connecting YingXao to WiFi, users will go to the signup page to create their own account. The credentials for each account should not be empty. The username name should be unique, and the password is required to be longer than 6. Hints will be popped out if any of them fails to meet the requirement.

After signing up or login through existing accounts, the display page will allow users to choose the plant. One thing to mention is that the plant choosing activity could be put in a separate page if a commercial plantInfo database is built. According to our experimental plantInfo database, we combine the plant choosing page and the data displaying page to make it easier to read.

Besides choosing the plant type, the displaying page shows users the current type of the plant they chose, the real-time sensor data and the desired data of humidity, sunlight, moisture, and temperature for the chosen plant. The real-time sensor data will update once the data in the firebase changes.
